# 題材の言語 MiniRuby を設計する

処理系を作るには、まず「何を実行する処理系なのか」を決めなければなりません。つまり**言語そのものの仕様**を決めるのが最初の仕事です。この章では、本書を通じて作り続ける小さな言語 **MiniRuby** を設計します。欲張らず、しかし言語処理系の本質的な要素は一通り味わえるように、機能を選んでいきましょう。

## どこまで作るかを決める

言語設計でいちばん難しいのは「足し算」ではなく「引き算」です。あれもこれもと機能を足すと、処理系はすぐに手に負えない大きさになります。入門の題材としては、**「実装してみると面白い仕組み」を含みつつ、全体を見通せる小ささ**が理想です。

MiniRuby には、次の 4 つの機能群だけを入れることにします。これは課題のブリーフで挙げられた「数値計算・条件分岐・ローカル変数・関数」に対応します。

1. **数値計算** ── 整数の四則演算と比較。
2. **制御構造** ── 条件分岐の `if` / `else` と、繰り返しの `while`。
3. **ローカル変数** ── 変数への代入と参照。
4. **関数** ── 関数の定義と呼び出し（再帰も含む）。

逆に、最初は次のものを**あえて入れません**。文字列・配列・オブジェクト・クラス・例外・クロージャ・GC ── これらはすべて応用編で改めて取り上げます。基礎編の目標は「**この 4 機能だけの言語を、構文解析から仮想マシンまで完全に動かす**」ことです。土台が小さいほど、各工程の役割がくっきり見えます。

> [!TIP]
> 言語を作るときは、最初に「**動く最小の言語**」を完成させ、そこへ機能を一つずつ足していくのが鉄則です。いきなり大きな言語を目指すと、どの工程でつまずいているのか分からなくなります。本書もこの方針です。

## MiniRuby の見た目

百聞は一見にしかず。MiniRuby のプログラム例を見てみましょう。フィボナッチ数を計算する、おなじみのプログラムです。

```ruby
def fib(n)
  if n < 2
    n
  else
    fib(n - 1) + fib(n - 2)
  end
end

puts fib(10)
```

これを実行すると `55` が表示されます。見ての通り、構文は Ruby のごく一部をそのまま借りています。Ruby を知っていれば追加で覚えることはほとんどありません。Ruby を知らなくても、この章を読めば MiniRuby は理解できます。

ホスト言語に Ruby を使い、題材も Ruby 風にするのには理由があります。第一に、Ruby はこうした処理系の記述に向いていること。第二に、できあがった MiniRuby のプログラムを**本物の Ruby でもそのまま動かして答え合わせ**ができることです。MiniRuby は「Ruby のうんと小さな部分集合」になるよう設計してあります。

## 言語仕様を言葉で決める

「Ruby に似ている」だけでは処理系は作れません。**何が書けて、何が書けないか**、そして**それぞれがどう振る舞うか**を、はっきり決める必要があります。これを **言語仕様（language specification）** と呼びます。MiniRuby の仕様を、データの種類・式・文・関数の順に決めていきましょう。

### 値の種類

MiniRuby が扱える値は、当面 **整数（integer）だけ**にします。`1`、`42`、`-3` のような値です。小数も文字列も真偽値も、最初は持ちません。

「真偽値がないのに、どうやって `if` の条件を判定するの？」と思うでしょう。ここは Ruby の流儀を借りて、**条件は整数で表す**ことにします。比較演算 `<` や `==` は、成り立つとき `1`、成り立たないとき `0` を返すことにし、`if` は「条件が `0` でなければ真」と解釈します。これは C 言語と同じ割り切りで、真偽値型を導入する手間を省けます。

> [!NOTE]
> 「真偽値を整数で代用する」のは、言語を小さく保つための**設計上の割り切り**です。本物の言語では `true`（真）／`false`（偽）／`nil`（「値がない」ことを表す空値）といった専用の値を持つのが普通で、応用編「値の表現とコンテナ」でこの拡張を扱います。小さく始めて後で広げる、という本書の方針の一例です。

### 式 ── 値を生むもの

**式（expression）** とは、評価すると値になる構文要素のことです。MiniRuby の式は次の通りです。

- **整数リテラル**：`0`、`123` など。
- **変数参照**：`x`、`total` など、ローカル変数の名前。
- **二項演算**：`a + b`、`a - b`、`a * b`、`a / b`（四則演算）と、`a < b`、`a > b`、`a == b`（比較）。
- **関数呼び出し**：`fib(10)`、`add(x, 3)` など。
- **括弧でくくった式**：`(a + b) * c` のように、計算の順序を明示するもの。

演算子の **優先順位（precedence）** ── どの演算を先に計算するか ── は数学の常識に合わせます。`*` と `/` は `+` と `-` より強く結びつき、比較演算はそれらより弱い、という順です。つまり `1 + 2 * 3 < 10` は `(1 + (2 * 3)) < 10`、すなわち `7 < 10` と解釈されます。また同じ強さの演算子は **左結合（left-associative）**、つまり `1 - 2 - 3` は `(1 - 2) - 3` と左から計算します。

### 文 ── 動作を起こすもの

**文（statement）** は、プログラムの実行を一歩進める単位です。MiniRuby の文は次の 4 つです。

**代入文**は、変数に値を入れます。

```ruby
x = 1 + 2
```

変数は事前の宣言が不要で、初めて代入したときに作られます（Ruby と同じ）。

**if 文**は条件分岐です。`else` は省略できます。

```ruby
if x < 10
  y = x * 2
else
  y = 0
end
```

**while 文**は繰り返しです。条件が `0` でないあいだ、本体の文を繰り返し実行します。

```ruby
while i < 6
  i = i + 1
end
```

**式文**は、式をそのまま文として置いたものです。とくに関数呼び出しは、値を捨てて副作用（画面出力など）のためだけに使うことがあります。

```ruby
puts x
```

ここで `puts` は、引数の値を画面に表示する**組み込み関数（built-in function）** です。組み込み関数とは、ユーザーが定義したのではなく、処理系自身があらかじめ用意している関数のことです。MiniRuby では当面 `puts` だけを用意します。

> [!NOTE]
> Ruby では「すべてが式」で、`if` 文すら値を返します。MiniRuby の実装もこの性質を取り入れ、AST インタプリタの章からは代入も `if` も「値を返す式」として一本の枠組みで扱います。ただし、説明のしやすさのため、この章では「文」と「式」を分けて語ります。両者の境界は言語設計上の選択であり、唯一の正解はありません。

### 関数 ── 処理に名前をつける

**関数（function）** は、ひとまとまりの処理に名前と引数をつけて、何度でも呼び出せるようにする仕組みです。MiniRuby の関数定義は `def` で始まり `end` で終わります。

```ruby
def add(a, b)
  a + b
end
```

関数の **本体（body）** は文の並びで、**最後に評価した式の値が戻り値**になります（Ruby には明示的な `return` もありますが、MiniRuby では用意しません ── 戻り値はつねに最後の式の値です）。引数 `a`、`b` は、その関数の中だけで使えるローカル変数として扱われます。

関数の中からは、自分自身を含む他の関数を呼び出せます。先ほどの `fib` のように**再帰呼び出し（recursion）** ── 関数が自分自身を呼ぶこと ── も当然できます。再帰が動くということは、処理系が**関数呼び出しごとに独立した変数の置き場所を用意できている**ことを意味します。この「置き場所」をどう管理するかが、後の章の山場のひとつになります。

## スコープ ── 変数が見える範囲

複数の関数が同じ名前の変数を使っても混ざらないように、変数には**見える範囲**があります。これを **スコープ（scope）** と呼びます。MiniRuby のスコープ規則はとても単純です。

- **ローカル変数は、それが属する関数の中でだけ有効**。
- 別の関数からは、たとえ同じ名前でも、その変数は見えない。
- 関数の引数も、その関数のローカル変数として扱う。

たとえば次のコードでは、`add` の中の `a` と、トップレベルの `a` は**まったく別の変数**です。

```ruby
def add(a, b)
  a + b
end

a = 100
puts add(1, 2)   # => 3
puts a           # => 100（add の中の a には影響されない）
```

このように、各関数呼び出しは自分専用の変数の領域を持ちます。この領域を **環境（environment）** や **フレーム（frame）** と呼び、関数を呼ぶたびに新しく作られ、関数を抜けると捨てられます。グローバル変数や入れ子の関数（クロージャ）といった、より複雑なスコープは応用編で扱います。

> [!IMPORTANT]
> 「変数がどこで見えるか」を決めるスコープ規則は、言語設計の重要な選択肢のひとつです。MiniRuby は「関数ごとに閉じる」という最も単純な規則を採用します。これだけでも再帰や関数の独立性はきちんと表現でき、処理系の基本構造を学ぶには十分です。

## 文法を記号で書き表す

最後に、ここまで言葉で決めてきた文法を、もう少し形式的に書いておきましょう。プログラミング言語の文法は、**BNF（Backus–Naur Form）** と呼ばれる記法でよく表現されます。BNF では、「`A` は `B` または `C` で構成される」といった規則を `A ::= B | C` のように書きます。MiniRuby の文法を BNF 風に表すと、おおよそ次のようになります。

```
program    ::= statement*
statement  ::= def | assign | if | while | expr
def        ::= "def" IDENT "(" params ")" statement* "end"
params     ::= ε | IDENT ("," IDENT)*
assign     ::= IDENT "=" expr
if         ::= "if" expr statement* ("else" statement*)? "end"
while      ::= "while" expr statement* "end"

expr       ::= comparison
comparison ::= add (("<" | ">" | "==") add)*
add        ::= mul (("+" | "-") mul)*
mul        ::= primary (("*" | "/") primary)*
primary    ::= INT | IDENT | call | "(" expr ")"
call       ::= IDENT "(" args ")"
args       ::= ε | expr ("," expr)*
```

ここで `IDENT` は変数名や関数名（識別子）、`INT` は整数、`*` は「0 回以上の繰り返し」、`?` は「省略可能」、`ε` は「空（何もない）」を表します。`comparison` → `add` → `mul` → `primary` という階層が、そのまま先ほど決めた**演算子の優先順位**を表現していることに注目してください。下にある（`primary` に近い）規則ほど、強く結びつきます。この文法は次章の構文解析でそのまま使います。

この BNF の書き方や、文法から構文解析器を導く理論は、ドラゴンブックをはじめとする定番教科書[Aho et al., 2006](#cite:aho2006)で体系的に学べます。本書では「文法を決めて、それを木に変える」という実用の流れを優先します。

---

これで MiniRuby の仕様が固まりました。扱う値は整数だけ、四則演算と比較、条件分岐と繰り返し、ローカル変数、関数定義と再帰呼び出し ── 小さいながらも、言語処理系の基本要素を一通り含んでいます。次章では、この文法に従って、ソースコードの文字列を抽象構文木へと変換する **構文解析** に取り組みます。
